Desenvolvendo classe Entidade Livro e Usuário

Os Objetos são classes que representam algo do mundo real, com suas propriedades. Por exemplo :um usuário, um livro, um produto, etc.

Nesse projeto vemos dois objetos (Livro , Usuário), cada um com suas propriedades ( dados ) para o desenvolvimento da aplicação.

Desenvolvendo Classe Context - Configuração para o banco de dados

Classe Context, realizamos as configurações para o banco de dados, podemos também implementar a estrutura para as funcionalidades do Entity Framework para o modelo de dados, fazemos essa configuração dos dados fazendo uso da Fluent Api.

Configurações da Classe Program

A Classe Program é a classe principal, onde instanciamos rotas, autenticações, autorizações, etc.

Configurações de Serviços / Adicionando Cors ( Restrição de política de mesma origem)/ Adicionando JWT Bearer

Migrations & Banco de Dados

Fazendo uso da ferramenta Entity Framework Tools, realizamos migração dos dados dos objetos modelos (User, Book). O recurso de migração diponibilizado pela EFCore permite criar novas entidades, propriedades, que podem ser adicionadas, atualizadas, ou removidas e os esquemas de banco de dados que será utilizada pela aplicação de forma sincronizada com o modelo de dados da aplicação.



Interfaces

Classe Interface: interface de visualização de dados referenciados (métodos CRUD -Create/Read/Update/Delete).


Repositórios

Camada responsável por trabalhar com a fonte de dados da aplicação , manipulação da lógica de acesso dos dados que será chamado na program fazendo uso da interface. Por meio dessa classe que se faz verificações dos dados consultados, existentes no banco de dados.



Classes Controles

Controle de acesso a API por meio dos endpoints, realizando métodos para criar, listar, atualizar, deletar dados de um objeto. O controle de acesso por meio dos endpoints é importante para restringir o acesso a API. Nos Controladores se encontra a regra de negócio implementando a API a parte de segurança por meio e autorizações e autenticações.

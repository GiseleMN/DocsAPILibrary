Descrição Geral

    Esse projeto estará realizando uma simulação de livraria, na qual terá duas entidades, sendo a classe de usuário(users), e a classe de livros( book). Cada classe com suas propriedades específicas, sendo que :


na classe usuário (User):

  • UserId( identificador gerado pelo banco de dados)
  • Propriedade email( email de cadastro para acesso)
  • Propriedade Password ( senha do cadastro de acesso)

na classe livro (Book):

  • BookId(identificador gerado pelo banco de dados)
  • Propriedade Title ( Título do livro)
  • Propriedade NumberOfPages ( Quantidade de páginas que o livro possui)
  • Propriedade Availability ( verifica a disponibilidade do livro sim/não)

Desenvolvendo Classe Context - Configuração para o banco de dados

    Configuração para o banco de dados, classe onde você estrutura as funcionalidades do Entity Framework para o modelo de dados, nessa classe você configurar os dados para a Fluent Api.
    A Fluent Api serve para complementar ou substituir as convenções pré definidas ou não, tendo como resultado final uma chamada de método encadeados como resultado final. Podemos sobrescrever os dados chamando o método OnModelCreating (modelbuilder).

Migrations & Banco de Dados

    Fazendo uso da ferramenta Entity Framework Tools, realizamos migração dos dados dos objetos modelos (User, Book). O recurso de migração diponibilizado pela EFCore permite criar novas entidades, propriedades, que podem ser adicionadas, atualizadas, ou removidas e os esquemas de banco de dados que será utilizada pela aplicação de forma sincronizada com o modelo de dados da aplicação.


     Modelo entidade User: após realizar a migração vemos pela imagem os dados com uso de Data Annotations(anotações). Parâmetros imposto na classe modelo usuário, limitando a quantidade de caracteres , tipo do dado, que será implementado no banco de dados.
    Modelo entidade Book: após realizar a migração, vemos os dados sendo implementados sem o uso de Data Annotations, a própria ferramenta implanta o tipo de dados a ser implementado no dados, porém vemos que não foi imposto um limite de valor de caracteres. ( Nesse caso não é uma boa prática) Nesse caso você pode realizar atualização dos por meio da Fluent Api.
